# 每日阅读简报 (Daily Brief)

## 流程:

### 1. 兴趣发现
分析用户工作区以识别兴趣点：
- 当前项目和关注领域
- 使用的技术和工具
- 经常提及的话题
- 感兴趣的行业或领域
- 个人目标和计划

参考指令: 这里的逻辑已委托给 `interest-analyzer` Agent。

### 2. 新闻收集
使用 `search_web` 工具搜索发现的兴趣相关的最新新闻，并调用 `news-curator` Agent 进行策划。

**关键要求：**
- 只包含**最近 7 天内**的新闻
- 使用日期过滤器进行搜索
- 验证发布日期
- 过滤出高相关性和可操作的内容
- 包含原始发布日期
- 解释每条新闻对用户的意义

### 3. 简报生成
将策划的新闻格式化为每日简报：
- 按相关性和优先级分组
- 突出显示发布日期
- 为每条信息添加个人上下文
- 建议可能采取的行动
- 突出显示紧急/有时效性的项目

### 4. 输出组织
将简报保存至 `life/daily-brief/YYYY-MM-DD-brief.md`，包含：
- 按主题划分的清晰章节
- 所有条目的发布日期
- "为什么这与你相关"
- 建议的行动
- 顶部的快速摘要
- **必须使用中文撰写**

## 主要功能:
- **个性化**：基于你文件中的真实兴趣
- **最新**：仅包含最近 7 天的新闻
- **已验证**：所有日期在包含前均已检查
- **可操作**：包含你可以用这些信息做什么
- **上下文化**：解释与你特定工作/兴趣的相关性

## 重要提示:
- **切勿**包含过时的新闻（超过 7 天）
- **始终**验证发布日期
- **始终**使用带日期过滤的网络搜索
- **始终**解释个人相关性
- **专注于**可操作的信息
