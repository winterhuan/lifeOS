# 周度复盘 (Weekly Check-in)

这是一个交互式的 Skill，分阶段执行。

## 流程:

### 1. 上下文发现与指标识别
### 1. 上下文发现与指标识别 (Context First)
- **第一步**：全面分析项目上下文：
    - 读取 `CLAUDE.md` 了解用户和项目。
    - 扫描 `business/`, `docs/` 或类似文件夹以获取上下文。
    - 检查 `metrics/metrics-history.md` (如果存在) 以了解历史跟踪记录。
    - **严禁使用通用模板**。必须基于此特定用户的项目类型来发现相关指标。

- **指标识别任务**：
    1.  **识别用户类型**：
        - 创作者 (Creators): 关注订阅者、浏览量、收入等。
        - SaaS: 关注 MRR、用户数、流失率、增长率等。
        - 开发者 (Developers): 关注提交数、PR、Star 数、下载量等。
        - 学生 (Students): 关注课程完成度、成绩、项目进度等。
    2.  **推荐关键指标**：基于类型，推荐 3-5 个最相关的关键指标。
    3.  **生成提问**：输出一个适合直接向用户提问的 Markdown 列表，要求用户提供本周数据。
        - 格式示例：
          > 基于你作为[类型]的上下文，我们重点追踪：
          > 1. [具体指标 1] (上周: X):
          > 2. [具体指标 2]:
          > ...

### 2. 数据收集 (交互)
- 基于识别出的指标，向用户提问，要求输入本周数据。
- **等待用户回复**。

### 3. 分析与报告
- 读取历史数据：`metrics/metrics-history.md` (如果存在)。
- 调用 `metrics-analyst` Agent 进行深度分析。

### 4. 输出组织
- 更新历史数据：将新数据追加到 `metrics/metrics-history.md`。
- 保存详细报告：`metrics/weekly-report-YYYY-MM-DD.md`。
- 向用户展示关键见解摘要。

## 关键特性:
- **智能适配**：不使用通用模板，根据你的实际项目类型（创作者/开发者/SaaS等）定制指标。
- **数据驱动**：关注增长率和趋势。
- **可视化**：生成 ASCII/Mermaid 图表。
- **中文输出**：全流程使用中文。

## 重要提示:
- 专注于**可操作的见解**，而不仅仅是数据收集。
- 如果是第一次运行，自动创建必要的目录。
